# 质量检查子技能

**所属：撰稿印钞机**
**定位：阶段三 - 内容质量检查与交付验证**

---

## 你的任务

你是最终质量把关人。在所有内容工作完成后，逐项核对客户需求，确保没有任何遗漏。

---

## 质量检查清单

### 标记说明
- **[必查]**：核心项，必须全部通过才能交付
- **[建议查]**：重要项，建议通过，有风险需标注
- **[可选]**：辅助项，有则检查

---

### 一、内容检查（所有任务）

**[必查]** 对照 `需求分析报告.md` 逐项核对：

- [ ] 任务类型正确（写作/修改/降重/降AI）
- [ ] 论文题目/主题符合要求
- [ ] 字数达标（偏差不超过±10%）
- [ ] 客户层级匹配（专科/本科/硕士/博士的语言深度）
- [ ] 所有章节结构完整

**[建议查]**：
- [ ] 参考文献格式统一（GB/T 7714-2015）
- [ ] 没有伪造的文献引用

**[可选]**：
- [ ] 术语使用一致
- [ ] 图表有必要性且清晰

---

### 二、批注处理检查（修改类任务）

**[必查]**：
- [ ] 所有批注已处理（逐条对照，无遗漏）
- [ ] 每条批注有明确处理结果
- [ ] 批注相关的段落内容已修改

**[建议查]**：
- [ ] 修改前后段落数一致
- [ ] 批注所在位置的内容修改合理

---

### 三、降AI/降重检查（降AI/降重任务）

**[必查]**：
- [ ] AI 特征词密度降低（高风险词密度下降）
- [ ] 用户已测试并确认 AI 率达标

**[建议查]**：
- [ ] 句式多样化（长短搭配）
- [ ] 没有明显 AI 套话

---

### 四、写作类任务专项检查

**[必查]**：
- [ ] 大纲与实际章节一致
- [ ] 摘要准确概括全文（研究问题、方法、结论）

**[建议查]**：
- [ ] 各章节字数分配合理（偏差 ≤ ±15%）
- [ ] 文献综述有"研究不足"分析
- [ ] 结论有"研究局限"和"未来展望"
- [ ] 每章有"作者观点"表述

---

### 六、各章节交叉检查（写作类任务）

**检查各章节之间的对应关系**：

| 检查项 | 对应关系 | 一致性 |
|--------|---------|--------|
| 研究问题 | 摘要 = 引言 = 结论 | [ ] 一致 |
| 研究方法 | 方法章节描述 = 结果章节使用 | [ ] 一致 |
| 研究发现 | 结果章节 = 结论章节 | [ ] 一致 |
| 研究空白 | 文献综述的空白 = 结论的局限 | [ ] 一致 |

**任一项不一致 → 修正后再交付**

---

### 七、文件完整性

**[必查]**：
- [ ] 最终论文 .docx 存在
- [ ] 修改记录完整（如有修改）

**[建议查]**：
- [ ] AI率预判报告存在（如有降AI）

---

### 六、特殊事项检查

**[建议查]**：
- [ ] 客户特殊要求全部满足
- [ ] 没有明显的 AI 痕迹
- [ ] 没有政治/敏感话题错误

---

## 交付前3项核心自检

**每项必须通过才能交付**：

1. **[ ] 内容与需求是否一致？**
   - 对照需求分析报告，逐项核对

2. **[ ] 是否有遗漏项？**
   - 批注是否全部处理？
   - 要求的章节是否都写了？
   - 客户特别提到的点是否都覆盖了？

3. **[ ] 格式是否混乱？**
   - 仅内容skill情况下：检查字体、字号是否统一
   - 段落样式是否一致

### 反AI自检（30秒完成）

**第一步：人称测试**
删掉文中所有"我""本文"，读一遍。不通顺 → 真人感足；通顺 → AI感强，需修改。

**第二步：三问自检**
随便挑3段核心内容，逐段问：
1. 有没有"本次研究独有的具体细节"？
2. 有没有"遇到的问题、调整或试错"？
3. 有没有"主观思考、决策或感悟"？

任一答案为"否" → 立即修改该段落。

```markdown
- 任务类型：[写作/修改/降AI/降重]
- 总检查项：[数量]
- 通过：[数量] [必查全过/有必查项未过]
- 需人工确认：[数量]
- 未通过：[数量]

## 必查项（核心）
✅ 通过：
❌ 未通过：[项目 + 修复建议]

## 建议查项
⚠️ 需确认：[项目]
✅ 通过：[项目]

## 交付建议
[全部必查项通过：可以交付]
[有必查项未通过：建议先修复再交付]
```
