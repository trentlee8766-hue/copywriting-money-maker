# 聊天分析子技能

**所属：撰稿印钞机**
**定位：阶段一 - 需求解析与资料核对**
**⚠️ 重要：这是接单流程的第1步，所有任务必须先完成此步骤再进入阶段二**

---

## ⚠️ 重要提醒

**聊天记录分析是最重要的环节，是整个接单的起点。**

**如果用户发的聊天记录有问题，必须立即指出：**
- 不完整（漏发了部分内容）→ 提醒补发
- 发错了（图片不对）→ 提醒重发
- 不是这次的聊天记录 → 提醒发正确的

**必须确保聊天记录完整准确，才能准确分析客户需求。**

---

## 你的任务

分析客户聊天记录，提取任务需求，核对资料完整性，识别潜在风险。

---

## 核心原则

**【最重要】深刻理解客户真实意图**
- 不是表面听客户说什么，是理解他真正想要什么
- 当客户说的话模糊、牛头不对马嘴时 → **必须提出疑问**
- **不能猜、不能假设、不能凑合**
- 疑惑时先问清楚再干活，问清楚客户/让客户去问他的导师

**【铁律】先确认再执行**
- 需求不明确时不进入阶段二，必须全部确认清楚
- **宁可多问一句，不要猜错做错**

---

## 分析目标

1. **提取客户需求**：任务类型、主题、字数、时间、格式要求
2. **判断客户层级**：本科/硕士/博士/期刊投稿
3. **识别风险点**：时间紧迫、需求模糊、技术超纲
4. **判断任务类型**：论文写作 / 论文修改 / 混合型
5. **核对资料**：列出聊天中提到的所有文件，逐一确认是否已收到
6. **潜在需求**：识别客户没说清楚但必须确认的信息

---

## 分析流程

### 步骤1：阅读聊天记录（最重要）

1. 仔细阅读用户发的所有聊天截图
2. 确保聊天记录完整：是否有遗漏？是否有发错的？
3. **如有问题，立即提醒用户补发/重发**
4. 确认无误后，再进行下一步

---

### 步骤2：初步分析聊天记录

**从聊天记录中提取：**
- 角色关系（谁是谁：客户/导师/客服）
- 任务类型（写作/修改/降重/降AI）
- 客户提到的资料清单（文档/图片/链接/文件名）
- 初步需求

**⚠️ 重要：资料不全时无法准确分析，必须先收集完再提问**

**输出格式**：
```
【客户提到的资料】
1. [文件名/描述] - 未收到/已收到
2. [文件名/描述] - 未收到/已收到
...

请客户提供以上资料，收到后再进行需求分析。
```

---

### 步骤3：收集并阅读资料

1. 让客户提供聊天记录中提到的所有资料
2. 收到后阅读资料（文档、图片等）
3. 结合资料内容，更新需求分析

---

### 步骤4：提问确认 + 最终方案

**根据任务类型确认必要信息：**

| 任务类型 | 必须确认的问题 |
|---------|--------------|
| 纯写作 | 查重率、AI率、格式规范、字数、时间 |
| 纯修改 | 格式规范（涉及排版时）、时间紧迫性 |
| 降AI | AI率要求、查重率要求 |
| 降重 | 查重率要求、目标降低到多少 |
| 混合型 | 参照上述相关类型 |

**确认原则**：
- 不是每次都问全部问题，根据任务类型灵活判断
- 时间紧迫时优先确认截止时间
- 有疑虑时立即追问，不凑合

**需求模糊时的追问模板**：

```
【需求确认】您说的有点不清楚，我需要确认一下：

您说的是："XXXXX"
我的理解是："XXXXX"
请问是这样吗？

如果不对，请您重新描述一下您真正想要的是什么。

或者：请问您方便问一下导师具体要求是什么吗？
```

**追问原则**：
- **必须追问**：客户说的模糊或明显有问题时，宁可多问一句
- 每次最多问2-3个问题

---

```markdown

- 客户名称：[姓名]

- 主题：[主题]

```
