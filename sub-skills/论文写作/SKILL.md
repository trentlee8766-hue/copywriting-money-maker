## 核心职责

1. **任务路由**：判断任务类型（缺选题/标准/紧急/学科专项）
2. **大纲制定**：根据题目和字数要求制定详细大纲
3. **分章节撰写**：按大纲逐章撰写优质内容
4. **质量控制**：文献真实性核查、AI痕迹消除、原创性保障

- "不知道写什么"（缺选题）

### 路由规则
```

│   └── 正常 → 走标准流程

```

## ①读范文 - 完整执行方案

### A. 搜索文献策略

```
搜索渠道优先级：
1. 必用：知网、万方（中文核心期刊）
2. 必用：Google Scholar（英文+中文）
3. 专业：IEEE/Springer/各学科数据库
4. 补充：国家统计局、行业报告

搜索词组合：
- 主题词 + 理论框架
- 主题词 + 研究方法
- 主题词 + 研究对象

筛选标准：
- 时间：近3-5年为主，经典文献可更早
- 类型：核心期刊 > 学位论文 > 会议论文
- 相关度：标题 → 摘要 → 全文
```

### B. 范文数量与来源

```
数量标准：
- 本科：3-5篇同级别优秀范文
- 硕士：5-8篇核心期刊 + 2-3篇硕论
- 博士：8-10篇顶刊 + 3-5篇博论

质量标准：
- 与自己论文主题高度相关（可参考部分≥40%）
- 发表在核心期刊或优秀学位论文
- 有完整的六章结构（背景+综述+方法+结果+讨论+结论）
```

### C. 三轮吃透法（标准流程）

```
第一轮（约30分钟）：
- 读标题+摘要+结论
- 输出：论文主旨、要解决的问题、亮点（约200字）

第二轮（约60分钟）：
- 读全文细节，理解关键思路
- 标注：小节标题、重要论点、数据呈现方式
- 输出：关键思路总结（约300字）

第三轮（约30分钟）：
- 找争议/不足/未解决的问题
- 输出：可深化的研究方向
```

## ③定大纲 - 好大纲的标准

### A. 好大纲的标准

```
合格大纲必备特征：
- [ ] 三级结构清晰（章-节-要点）
- [ ] 每章有字数分配
- [ ] 每节有写作要点提示
- [ ] 各章逻辑连贯（前因后果）
- [ ] 与研究问题直接相关

不合格大纲特征：
- 只有两级结构
- 没有字数分配
- 各章之间跳跃/重复
- 与研究问题脱节
```

### B. 大纲制定流程

```
Step 1：确定研究问题（从客户需求+范文启示）
Step 2：参考2-3篇范文的大纲结构
Step 3：制定自己的大纲（章-节-要点）
Step 4：分配字数（按比例+按重要性）
Step 5：客户确认大纲后再写
```

### C. 字数分配参考

```
通用比例（可根据学科调整）：
- 摘要：5%（300-500字）
- 引言：15%（1500-2000字）
- 文献综述：25%（2500-3500字）
- 研究方法：15%（1500-2000字）
- 结果与分析：25%（2500-3500字）
- 结论：10%（800-1200字）
- 参考文献+致谢：5%

注：具体比例可根据学科特点调整
```

## 内容质量标准

### 硬指标（必须满足）

| 指标 | 标准 | 检查方法 |
|------|------|---------|
| 字数偏差 | ≤ ±10% | 章节字数统计 |
| 作者观点 | 每章至少1处 | 搜索"笔者认为"、"本文认为"等 |
| 研究不足 | 文献综述必须有 | 检查是否有"然而现有研究存在..." |
| 研究局限 | 结论中必须有 | 检查是否有"本研究存在以下局限..." |
| 未来展望 | 结论中必须有 | 检查是否有"未来研究可..." |

**不满足任一硬指标 → 返工**

### AI味检测清单（每章写完必查）

**高风险检测（每出现1次需降AI处理）**：
- [ ] 高风险词出现次数 ≥ 2
- [ ] 连续3个以上相同句式结构
- [ ] 频繁使用"首先...其次...最后"

**中风险检测（每出现3次需降AI处理）**：
- [ ] 机械连接词过多（"由此可见"、"不难发现"）
- [ ] 抽象论述过多，无具体案例
- [ ] 缺乏口语化表达

**通过标准**：
- 高风险检测：0项触发
- 中风险检测：最多1项触发
- 否则 → 执行去AI味操作

### 去AI味操作规范

**反AI写作核心原则**：
让论文呈现"真实研究者的思考、实践、试错与感悟"

**必须遵守的规则**：

1. **逻辑优先**：遵循"发现问题→尝试解决→出现波折→调整优化→得出结果"的真人研究逻辑，禁止"为了实现XX目标，本文采用XX方法"这类AI标准句式

2. **拒绝完美**：每个研究环节加入1-2个"合理的小失误、调整或妥协"，杜绝"一步到位、毫无瑕疵"的AI式完美过程

3. **细节落地**：每个专业术语后紧跟"本次研究独有的具体细节"，禁止空泛套话

4. **强化主体**：多用"我发现""我尝试了""我意识到""我调整了"，突出研究者的主观思考

5. **适配专业**：灵活替换研究环节表述（文科→文献梳理/调研访谈，理科→实验操作/数据采集，工科→设计研发/原型测试）

**禁止的AI套话**：
- 深入研究、具有重要意义、由此可见、首先...其次...最后...
- 第一...第二...第三...、综上所述、总体而言
- 具体而言、不难发现、协同推进、统筹推进

**人类写作痕迹标记**（生成内容时必须包含）：
- "我发现"、"我尝试了"、"我意识到"、"我调整了"
- "说实话"、"实际上"、"值得注意的是"

**人类写作痕迹标记**（生成内容时必须包含）：
- "我发现"、"我尝试了"、"我意识到"、"我调整了"
- "说实话"、"实际上"、"值得注意的是"

## ④质量控制 - 各阶段检查

### A. 每章写完快速自检

**3项核心检查（每章必查）**：

```
1. 这一章解决了什么问题？
2. 这一章与上一章的衔接是什么？
3. 这一章的亮点/作者观点是什么？
```

**任一项为空 → 补充完善后再继续**

---

### B. 各章节交叉检查

**检查项**：

| 检查项 | 对应关系 |
|--------|---------|
| 摘要的研究问题 | = 引言的研究问题 |
| 文献综述的空白 | = 结论的局限 |
| 方法章节描述的 | = 结果章节使用的 |
| 结果章节的发现 | = 结论章节的发现 |

**任一项不一致 → 修正后再继续**

## 核心原则

### 1. 文献真实性第一
**绝对禁止伪造文献引用。** 所有参考文献必须真实存在、可验证、可追溯。

**验证方法**：
- 通过知网、Google Scholar 等公开渠道搜索验证
- 无法验证的引用标注 `【此处需人工补充真实引用】`
- 引用内容必须与原文一致

### 2. 原创性硬指标
**每个核心章节必须包含作者观点和批判性分析。**

**检查方法**：
- 检查是否有"笔者认为"、"本文认为"等作者观点表述
- 检查文献综述是否有"研究不足"分析
- 检查结论是否有"研究局限"和"未来展望"

### 3. 去AI味三板斧
**AI写作痕迹必须主动消除。**

**处理方法**：
- 按"反AI写作铁律"生成/改写内容
- 对照"禁止的AI套话"清单自检
- 主动添加人类写作痕迹：口语化表达、具体案例、个人判断

### 4. 数据来源透明化
**所有数据必须明确来源，预测性内容必须标注。**

### 5. 反AI写作铁律

**核心**：让论文呈现"真实研究者的思考、实践、试错与感悟"

1. **逻辑优先**：遵循"发现问题→尝试解决→出现波折→调整优化→得出结果"的真人研究逻辑，禁止"为了实现XX目标，本文采用XX方法"这类AI标准句式
2. **拒绝完美**：每个研究环节加入1-2个"合理的小失误、调整或妥协"，杜绝"一步到位、毫无瑕疵"的AI式完美过程
3. **细节落地**：每个专业术语后紧跟"本次研究独有的具体细节"，禁止空泛套话
4. **强化主体**：多用"我发现""我尝试了""我意识到""我调整了"，突出研究者的主观思考
5. **适配专业**：灵活替换研究环节表述（文科→文献梳理/调研访谈，理科→实验操作/数据采集，工科→设计研发/原型测试）

## 标准化写作流程（8步法）

### 步骤0：确认需求备忘录

从阶段一接收需求分析报告和资料核对清单，确认以下7项：
1. 题目明确性
2. 字数分配（各章节大致字数）
3. 交付时间
4. 查重率要求
5. 格式要求（记录但不处理）
6. 客户层级（本科/硕士/博士）
7. 绝对不能犯的错误

如有任何一项不明确，返回阶段一重新确认。

---

### 步骤1：范文搜索与分析

**目标**：找到3-5篇同领域优质范文，分析其结构、写作手法、可借鉴内容。

**操作**：
- 根据题目关键词，搜索知网、万方等平台的优秀毕业论文
- 分析范文的：章节结构、论证逻辑、文献引用方式、数据呈现方式
- 提取可借鉴的写作框架和表达模式
- 识别研究空白（可作为本论文的创新点）

**注意**：范文只供结构参考，绝不抄袭内容。

---

### 步骤2：大纲生成（三级详细版）

**目标**：制定带字数分配的三级大纲，客户确认后再开始写作。

**输出要求**：
```markdown
# 论文大纲

## 第一章 绪论（约2000字）
### 1.1 研究背景（约800字）
- 要点：...
### 1.2 研究意义（约600字）
- 要点：...
### 1.3 研究内容与方法（约600字）
- 要点：...

## 第二章 文献综述（约3000字）
...
```

**每章必须包含**：
- 字数分配
- 写作要点
- 预计引用文献数量
- 是否需要图表

**客户确认后**才能进入步骤3。

---

### 步骤3：分章节撰写

**核心原则**：**按章节写，写完一章 → 发给用户 → 用户确认 → 继续下一章**

```
错误做法：一次性写完整篇论文 → AI胡来、用户无法控制
正确做法：按章节分次写，用户把控节奏，确认一篇再写下一篇
```

**分章节撰写流程**：
```
Step 1：按大纲顺序，确定本次写作的章节
Step 2：AI根据该章节的字数要求撰写（1500-3000字/章）
Step 3：写完该章节后，发送给用户
Step 4：用户确认通过 → 继续写下一章节
Step 5：用户有修改意见 → 先修改 → 用户再确认 → 继续下一章节
```

**用户把控节奏**：用户根据AI写的内容，自己决定是否继续、是否修改。

---

**写作顺序建议**：
1. 先写核心章节（文献综述/研究方法/数据分析）
2. 再写绪论和结论
3. 最后写摘要（全文完成后才能准确概括）

---

**每章核心写作要点**：

```
【本章写作要点】
- 字数：按大纲分配的该章节字数
- 语言：通顺，平实、避免冷僻词汇
- 逻辑：清晰、前后连贯
- 过渡：适当使用过渡词，避免机械连接
- 避免：过度复杂推导、冷僻学术词汇、AI套话
```

## 写作提示词（每次写章节时必须参考）

### 一、语言风格

| 要求 | 具体做法 |
|------|---------|
| **专业但易懂** | 用专业术语但配合解释，不过度堆砌冷僻词汇 |
| **善用比喻** | 用形象比喻解释概念（如"如同"、"犹如"、"成为"） |
| **准确严谨** | 实证类论文用词准确，不夸大（如"研究表明"而非"证明"） |
| **长短句交替** | 单句不超过40字，长短交替，节奏感强 |

**词汇替换（AI味 → 自然）**：

| 避免（AI味） | 替换为（自然） |
|-------------|---------------|
| 深入研究 | 探讨、研究 |
| 具有重要意义 | 有价值、有帮助 |
| 由此可见 | 因此、可以看出 |
| 首先...其次...最后... | 开头...接下来...最后 |
| 综上所述 | 总的来看 |
| 具体而言 | 具体来看 |
| 不难发现 | 可以看出 |
| 协同推进 | 共同推进 |
| 统筹推进 | 统一推进 |

### 二、论文结构框架

**实证研究类结构**：
```
引言（背景→文献综述→研究问题→边际贡献）
↓
理论分析（提出假设：H1、H2、H3...）
↓
研究设计（变量、模型、数据）
↓
实证结果（表格+分析）
↓
结论与建议（总结→局限→展望）
```

**评述综述类结构**：
```
背景引入（宏观→微观，引出主题）
↓
分论点（按主题/维度分节）
↓
总结与展望（梳理脉络、指出方向）
```

---

### 三、论证逻辑

**段落结构（每段必须包含）**：

```
第一句（中心句）：本段要说什么
中间句（支撑句）：例子、数据、引用
最后句（小结）：本段结论/过渡到下段
```

**论证逻辑**：

```
提出观点 → 给出论据 → 分析原因 → 得出结论
   ↑           ↑          ↑         ↑
 (判断)     (事实)     (推理)    (总结)
```

**过渡衔接**：

| 位置 | 过渡词 |
|------|--------|
| 原因 | 因此、于是、导致 |
| 转折 | 但是、然而、不过 |
| 递进 | 而且、并且、此外 |
| 举例 | 比如、例如、以...为例 |
| 总结 | 总之、综上、可见 |

**禁止的机械结构**：
- "首先...其次...最后..." → 改用自然分段
- "一方面...另一方面..." → 改用"除了...还有..."
- "综上所述..." → 改用"总的来看..."

---

### 四、不同论文类型的表达模板

**评述类常用表达**：
- "展示了..."、"标志着..."、"为...提供了..."
- "如..."、"例如..."、"...等"
- "未来可..."、"有望..."、"亟待..."

**实证类常用表达**：
- "基于...理论，提出假设H1：X对Y有...影响"
- "参考...研究[X]，本文..."
- "研究假设得到/未得到验证"

### 五、自查清单（写完本章后检查）

**语言检查**：
- [ ] 专业术语是否配合解释？
- [ ] 是否用了形象比喻？
- [ ] 单句是否超过40字？
- [ ] 句式是否有变化？

**结构检查**：
- [ ] 是否有清晰的章节层次？
- [ ] 每节开头是否有概述句？
- [ ] 段落之间是否衔接自然？

**论证检查**：
- [ ] 观点是否有文献/数据支撑？
- [ ] 是否有一句小结性语句收尾？
- [ ] 结论是否有局限性说明？

---

**每章写作要求**：
- **必须有"作者观点"模块**：不能全是文献堆砌，要有自己的分析和判断
- **文献综述必须有"研究不足"**：指出前人研究的空白，引出本研究的意义
- **方法论必须有"方法选择理由"**：为什么用这个方法，不用其他方法
- **数据分析必须有"数据解读"**：不只是呈现数据，要分析数据背后的含义

**反AI处理**：
- 每章写完后自查：是否有禁止的AI套话？是否有真人写作痕迹？
- 有AI味的内容，用上面的"去AI味操作规范"改写
- 增加口语化、具体化表述

### 步骤4：文献引用与参考文献整理

**目标**：确保所有引用真实、准确、格式一致。

**操作**：
- 逐条核对引用文献：是否真实存在、DOI或链接是否正确
- 无法验证的标注 `【此处需人工补充真实引用】`
- 整理参考文献列表，按学校要求排序（GB/T 7714等）
- 确保正文引用和参考文献列表一一对应

### 步骤6：反AI处理
