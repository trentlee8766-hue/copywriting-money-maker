# 商业计划书写作子技能

**版本：1.0.0**
**更新日期：2026年5月1日**
**核心定位：帮客户把"商业想法"变成"专业BP"**

---

## 适用场景

- 客户有商业想法但不知道怎么写成BP
- 客户有项目需要融资
- 客户需要向投资人展示项目
- 其他需要商业计划书的场景

**注意**：本技能只在用户明确提到上述触发词时调用，不影响论文类业务的正常流程。

---

## 核心痛点

**客户的问题**：
- "我有XXX想法，想融一笔钱"
- "我知道市场有这个机会，但不知道怎么说清楚"
- "我有技术/产品，不知道怎么包装成投资人想看的"

**本技能的价值**：
把散乱的商业想法 → 结构化的投资人想看的BP

---

## 工作流程（三阶段）

### 阶段一：需求解析与商业梳理

**目的**：挖掘客户真实想法，梳理完整商业逻辑

**步骤**：

1. **挖掘基本信息**
   - 项目名称/定位
   - 创始人背景
   - 目标融资金额
   - 当前进展（想法/原型/已有收入）

2. **深度挖掘商业模式**（按顺序逐个确认）
   ```
   【必问7问】
   1. 你的客户是谁？（谁会买单）
   2. 他们有什么痛点？（为什么需要你）
   3. 你提供什么解决方案？（产品/服务是什么）
   4. 怎么赚钱？（收入来源、定价）
   5. 怎么获客？（营销渠道、销售方式）
   6. 你的竞争对手是谁？（差异化在哪）
   7. 你需要多少钱？（用途、预期回报）
   ```

3. **行业分析**（根据客户提供的信息补充）
   - 行业规模（ TAM/SAM/SOM ）
   - 行业趋势
   - 政策环境

4. **竞争定位**
   - 直接竞争对手
   - 间接竞争对手
   - 自身差异化优势

5. **财务预测**（基于客户提供的数据）
   - 收入预测（3-5年）
   - 成本结构
   - 盈亏平衡点

**输出**：
- 商业梳理备忘录.md（客户确认商业模式）
- 需求分析报告.md（如需）

**确认节点**：客户确认商业模式梳理结果后再进入阶段二

---

### 阶段二：BP撰写

**目的**：按投资人关心的逻辑撰写完整BP

**标准章节结构**（按重要性排序）：

```
【必写章节】

1. 执行摘要（最后写）
   - 一句话定位
   - 核心痛点与机会
   - 产品/解决方案
   - 核心数据（已有成果）
   - 团队亮点
   - 融资需求与用途

2. 市场分析
   - 行业规模与增长
   - 目标市场细分
   - 竞争格局
   - 自身定位

3. 产品与服务
   - 核心功能
   - 解决痛点
   - 差异化优势
   - 进展与路线图

4. 商业模式
   - 收入来源
   - 定价策略
   - 单位经济模型
   - 可行性验证

5. 营销策略
   - 获客渠道
   - 转化漏斗
   - 客户留存

6. 团队介绍
   - 创始人背景
   - 核心成员
   - 团队优势

7. 融资计划
   - 本轮融资需求
   - 资金用途
   - 预期里程碑
   - 退出路径

【可选章节】（根据需要添加）

8. 运营计划
9. 财务规划
10. 风险分析
11. 附录
```

**撰写原则**：
- 每个章节都要回答投资人的核心问题
- 数据要自洽（前后一致）
- 优势要具体，劣势要诚实
- 避免空话、套话

**分步撰写机制**：
```
① 先写大纲 → 客户确认
② 逐章撰写 → 每章发给客户确认
③ 最后写执行摘要（基于全文）
```

**输出**：
- BP大纲.md（客户确认）
- 各章节草稿.docx（逐章确认）
- 最终BP.docx

---

### 阶段三：质量检查与交付

**目的**：确保BP专业、逻辑自洽、可说服投资人

**检查清单**：

| 检查项 | 标准 |
|--------|------|
| 逻辑闭环 | 痛点→方案→盈利→增长全链路通顺 |
| 数据自洽 | 所有数字前后一致，无矛盾 |
| 差异化突出 | 相比竞争对手优势明确 |
| 投资人视角 | 每个章节回答"为什么值得投" |
| 文档规范 | 格式统一、重点突出、可读性强 |

**交付文件**：
```
[项目名]_商业计划书_YYYYMMDD.docx
```

## 核心原则

1. **客户想法第一**：所有内容基于客户提供的想法，不凭空编造
2. **投资人视角**：每个章节思考"投资人在意什么"
3. **数据自洽**：所有数字必须前后一致，可验证
4. **诚实为本**：优势说足，劣势不隐瞒
5. **逻辑闭环**：痛点→方案→盈利→增长全链路通顺
