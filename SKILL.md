# 撰稿印钞机

**版本：5.6.0**
**更新日期：2026年4月30日**
**核心定位：专注内容。深度理解需求 + 优质内容输出。**

---

## 能力范围

| 能力 | 说明 |
|------|------|
| 需求解析 | 深度分析客户需求，核对资料完整性，识别风险点 |
| 论文修改 | 批注处理、降重、降AI、内容调整 |
| 论文写作 | 从头原创，按客户要求撰写完整论文（含反AI处理） |
| 质量检查 | 内容验证、文献真实性核查、交付前把关 |

直接描述需求即可：

- "写一篇会计专业的毕业论文"
- "这篇论文需要降AI"
- "处理一下导师的批注"

## 用户模式

根据用户习惯，自动调整确认节奏：

| 模式 | 触发关键词 | 确认频率 |
|------|-----------|---------|
| beginner | 新手、不熟悉、第一次 | 每步确认，附带详细解释 |
| standard | 无关键词 | 关键节点确认 |
| expert | 老手、熟悉、懂的 | 仅MANDATORY确认 |

**用户没说 → 默认 standard 模式**

## 核心铁律

1. **内容第一**：先搞定内容
2. **信息落盘，不靠记忆**：每个阶段结束必须把关键信息写入文件
3. **绝不伪造文献**：无法验证的引用标注 `【此处需人工补充真实引用】`
4. **不漏任何资料**：聊天记录中提到的每个文件都必须列入核对清单
5. **确认节点**：按 `config/confirmation_rules.json` 执行确认流程

---

## 工作流程（三阶段）

### 阶段一：聊天分析（第1步）

**目的**：深度理解客户需求，核对资料完整性

**⚠️ 重要**：这是接单流程的入口，所有任务必须先完成此步骤再进入阶段二

**步骤**：
1. 分析聊天记录，提取任务类型、主题、字数、时间、特殊要求
2. 列出资料核对清单，逐项确认
3. 读取客户提供的文件，检查匹配情况
4. 风险评估（时间紧迫/需求模糊/资料缺失）

**输出**：
- 需求分析报告.md
- 资料核对清单.md（全部确认后才能进入下一阶段）

**调用**：`sub-skills/聊天分析/`

---

### 阶段二：内容处理

**目的**：完成纯内容工作

**论文修改**（调用 `sub-skills/论文修改/`）：
1. 提取修改要求（批注/图片/文本）
2. 分类处理（删除/替换/按反AI规范生成插入）
3. 内容调整（结构、逻辑、补充）

**论文写作**（调用 `sub-skills/论文写作/`）：
1. 制定大纲（客户确认后再写）
2. 分章节撰写（按反AI写作规范）
3. 内容审核修改

**输出**：内容已确认的文档.docx

**关键**：
- 修改时保持原始段落格式不变（不增删段落，不改样式）
- **先复制副本，在副本上修改，原始文档保持不动**
- 严禁直接在原文档上修改
- 严禁用 `deepcopy` 复制段落到新Document

---

### 阶段三：质量检查与交付

**目的**：逐项核对，确保无遗漏 → 调用 `sub-skills/质量检查/`

**检查清单**：
- [ ] 任务类型是否正确处理
- [ ] 字数是否符合要求
- [ ] 章节结构是否完整
- [ ] 文献引用是否真实（无法验证的已标注）
- [ ] 批注是否全部处理（如需要）
- [ ] 内容是否通顺、逻辑是否连贯
- [ ] 文件完整性（所有要求文件是否已交付）

**输出**：质量检查报告.md

## 信息落盘协议

| 阶段 | 落盘文件 |
|------|---------|
| 阶段一完成 | 需求分析报告.md、资料核对清单.md |
| 阶段二完成 | 内容确认版文档.docx、修改记录.md |
| 阶段三完成 | 质量检查报告.md |

下一个阶段开始时，先读取上一个阶段的落盘文件，不依赖对话记忆。

## 交付文档命名规范

**最终交付文档**：
```
[论文名]_内容版_YYYYMMDD.docx
例：小学教育论文_内容版_20260430.docx
```

**中间文件**：
```
[论文名]_阶段_说明.docx
例：小学教育论文_第一轮降AI.docx
     小学教育论文_批注处理后.docx
```

**输出目录结构**：
```
/output/
  ├── [论文名]_内容版_日期.docx  ← 最终交付
  ├── 需求分析报告.md
  ├── 资料核对清单.md
  ├── 修改记录.md
  └── 中间文件/  ← 可选保留
```
